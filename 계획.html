프로젝트 계획서: Simple Board Prototype

기술 스택

HTML/CSS
웹페이지의 구조와 시각적 스타일을 정의합니다.
TypeScript
JavaScript에 타입을 추가하여 안정적인 개발을 가능하게 합니다.
Node.js & Express
서버 사이드 로직을 처리하고 RESTful API를 구축합니다.
Neon (PostgreSQL)
클라우드 기반의 서버리스 관계형 데이터베이스로 데이터를 저장합니다.

시각적 디자인 샘플 (5개)

1. Clean Minimalist

여백을 강조하고 얇은 테두리와 파란색 강조색을 사용한 깔끔한 디자인.

2. Modern Card-based

그림자와 둥근 모서리를 가진 카드 형태의 레이아웃과 굵은 타이포그래피.

3. Classic Forum

전통적인 테이블 기반의 게시판으로 밀도 높은 정보 전달에 최적화.

4. Soft Pastel

부드러운 파스텔 톤 색상과 둥글둥글한 버튼을 사용한 친근한 디자인.

5. High-Contrast Pro

어두운 헤더와 날카로운 경계선, 굵은 폰트를 사용한 전문적인 느낌.

테이블 목록

  • users: 회원 정보 (아이디, 비밀번호 등)
  • posts: 게시글 (제목, 내용, 작성자 등)
  • comments: 댓글 (게시글 ID, 내용, 작성자 등)
  • likes: 추천 (게시글 ID, 사용자 ID - 중복 방지)

CREATE TABLE 문

CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE posts (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE comments (
    id SERIAL PRIMARY KEY,
    post_id INTEGER REFERENCES posts(id) ON DELETE CASCADE,
    user_id INTEGER REFERENCES users(id),
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE likes (
    post_id INTEGER REFERENCES posts(id) ON DELETE CASCADE,
    user_id INTEGER REFERENCES users(id),
    PRIMARY KEY (post_id, user_id)
);
